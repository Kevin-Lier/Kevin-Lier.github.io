多页应用

  • 页面跳转====》返回html
  • 优点：首屏时间快，SEO（搜索引擎优化）效果好
  • 缺点：页面切换慢

单页应用

  • Vue项目为单页应用服务器渲染技术，不会每次切换都请求html文件
  • 页面跳转===》JS渲染
  • 优点：页面切换快
  • 缺点：首屏时间稍慢，SEO差

        axios.get('/news/index').then(this.getHomeInfoSucc)
    },

        res = res.data
        if(res.ret && res.data)
        {
            const data = res.data
            this.city = data.city
            this.swiperList = data.swiperList
            this.iconList = data.iconList
            this.recommendList = data.recommendList
            this.weekendList = data.weekendList
        }
    }
},
mounted () {
    this.getHomeInfo()
}

组件

每个组件就是页面上的一个区域

  • 全局组件:
	<ul>
		<todo-item v-bind:content="item"
					v-for="item in list">
		</todo-item>
	</ul>

	<script>
        var app = new Vue({
		el:"#app",
		data:{
			list:['第一课内容','第二课内容'],
			inputValue:''
		},
		methods:{
			handleBtnClick:function(){
				this.list.push(this.inputValue)
				this.inputValue=""
			}
		}
	})
	// 全局组件
	Vue.component("TodoItem",{
		props:['content'],
		template:"<li>{{content}}</li>"
	})
	</script>
  • 局部组件
	<script>
        var app = new Vue({
		el:"#app",
        components:{
            TodoItem: TodoItem
        },
		data:{
			list:['第一课内容','第二课内容'],
			inputValue:''
		},
		methods:{
			handleBtnClick:function(){
				this.list.push(this.inputValue)
				this.inputValue=""
			}
		}
	})
	//局部组件
	var TodoItem = {
		props:['content'],
		template:"<li>{{content}}</li>"
	}
	</script>

父组件给子组件传值：v-bind（简写为冒号:） 在子组件中需要接收，用props

子组件通过$emit事件触发向上一层触发事件，父组件监听，监听后可以带出子组件的内容

组件作为另一组件的子组件时应该用is属性标注

组件中data必须是一个函数返回一个对象，否则组件不能复用，即一个组件改变会影响到其他所有的组件

计算属性和侦听器

计算属性（computed）:用于简单运算 语法更简单

  • 计算属性拥有缓存机制，性能更好

  • 计算属性的getter和setter

    computed: {
  fullName: {
    // getter
    get: function () {
      return this.firstName + ' ' + this.lastName
    },
    // setter
    set: function (newValue) {
      var names = newValue.split(' ')
      this.firstName = names[0]
      this.lastName = names[names.length - 1]
    }
  }
}

    通过这样的方式可以改变firstname和 lastname

条件渲染

  • v-if
  • v-else 必须紧跟在v-if后，否则不会被识别
  • v-else-if
  • v-show 与v-if基本相同，不同在于v-show为初始渲染而v-if为条件渲染，因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。
